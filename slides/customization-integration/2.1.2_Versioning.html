# Nuxeo training - Nuxeo studio versioning
###Context

You just deployed your new document types and realised they do not show the proper information on screen.

It makes you think about an important feature: how may I cancel a change if someday I make a mistake ? Having a versioning tool would be appropriate. Fortunately, Studio integrates this feature natively. Let's see how.

###Nuxeo studio - Versioning #### Concepts - Save your project state - Revert to a previous state anytime - Release versions and deploy them easily - Simplified versioning tool: No branching and no conflict management
###Nuxeo studio - Versioning #### Managed in `Settings and versioning` > `Versioning` menu #### Three different stages - Commits - Preferably used for small changes - Do not hesitate to commit often so you can revert a change if needed without losing too much - Tags - When you feel the version is complete enough - Any commit can be tagged - Tags remain available even if you revert to a commit done before the tag - Snapshots - When you feel the version can be released for production - Snapshots can be downloaded and installed directly from the admin center
###Nuxeo studio - Versioning #### Management - Define your commit style from `Settings and versioning` > `Project settings` - On save: Automatic commit each time you press save on a screen - On build: When you create a tag or a snapshot- Manual: Only when you choose to - No branching and no conflict management implementation
###Practice Nuxeo studio #### Versioning - Check your commit style in your project settings - Make sure you have commited your changes so far - Create a tag from your commit

# Nuxeo training - Form layouts
### Context

So the metadata we want on screen for our custom document types is not displayed yet. What a shame… you should get it sorted out asap!

###What we want to do (1/3) ####At the end of the course: - Create a Book document with a custom creation form
###What we want to do (2/3) ####At the end of the course: - View the custom fields from the `Summary` tab
###What we want to do (3/3) ####At the end of the course: - Edit the custom fields from the `Edit` tab
### Layouts? #### Define… - What metadata is shown on screen - Where (on top, on bottom, to the left...) - When (when creating, editing, viewing a document...) - How (as a text field, text area, calendar…) ####Main layouts - Creation layout: Displayed when creating the document. - Edit layout: Displayed when going on the edit tab of a document. - View layout displayed on the «summary» tab. - Header layout: Displayed on top of the tabs when viewing a document in Nuxeo DM.
### Layouts in Nuxeo Studio #### How / where can I create them? - Directly in your document type - In the `Listing and views` > `Form layouts` menu in order to make it reusable for another document type. #### How to use them? - Head to the tab corresponding to the layout you wish to define - Define your table structure - Drag and drop widgets from the right side of the screen to a table cell
### Widget? - Used to display or fill in document metadata - Defines how the field will be shown to the user - Behaves differently in edit or view mode
Ex: For a calendar, in view mode, the calendar widget displays the date value, in edit mode, it displays a calendar to select a value
- Many different widgets types available: - Text: Simple input text field - Single User/group suggestion: Lists users and groups - Document suggest: Select another existing document - Vocabulary and Chained vocabularies: A list of values, on one or two levels - And many others...
### Practice (1/3) #### Form layouts - Add the ISBN field in the creation, view, and edit layouts of your `Book` document type - The field used for this widget will be book:isbn (that makes sense, right?)  - ISBN as label is really a good choice - The label doesn’t need to be translated - We want this field editable when we create the document, and then read only when viewing or editing the document 
### Practice (2/3) #### Try out your changes - Deploy your changes using hot reload - Create a new Book - Check that when creating it, you can enter the fields title, description and ISBN - When editing, the ISBN must be unmodifiable
### Practice (3/3) #### Some more improvements... - Add a widget for the `book:author` field in each layout - Make sure the subwidgets label is displayed - The subwidgets should be required when filling an author - Add a new widget for the `book:publicationDate` - This field should be: - Required and editable in creation layout  - Visible in the view layout - Not displayed in edit layout

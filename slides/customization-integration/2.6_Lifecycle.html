# Nuxeo training - Life cycle
### Context

Seeing what you have done with their book libraries, you are now kindly being requested to add more:

- A separate view of books still in library and borrowed when viewing a `BookLibrary`

- A listing of books from the same category when viewing a `Book`.

To achieve these, we will need to define states for our books…

###What we want to do ####At the end of the course - Create a life cycle for Book documents - Apply this life cycle to the Book document type
###Life cycle #### Concepts - Defines the state of a document - Defines allowed transitions from a state to another - Builtin core feature: All documents in Nuxeo have a life cycle and a state - Each document type can have its own life cycle
### Life cycle #### The trash folder The deleted state and the transitions delete and undelete are used to have the «trash» feature on the document whatever life cycle policy used in the document type. If you want to activate the trash feature on a document type that uses a custom life cycle policy, you must add those states and transitions to it.
### Practice (1/2) #### Life cycle - Create a new life cycle policy called BookLifeCycle - Initial state: "inLibrary" - Create another state "borrowed" - Create transitions to link each other - Do not name them - Save the life cycle. What are you seeing?
###Practice (2/2) #### Life cycle - Create the "deleted" state - Add a `delete` transition for each previously created state - Add an `undelete` transition going back to the inLibrary state - Apply this life cycle to the `Book` document type definition - Commit your changes - Deploy your changes using hot reload - Check the initial state by creating a new Book We will create an action to change the life cycle later

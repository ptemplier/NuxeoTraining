# Nuxeo training - Vocabularies
### Context

The "Best books" library is happy to see they can create their books as intended. But there is still something missing: they would like to be able to categorize their books into a list of predefined options. One more challenge coming in.

###What we want to do ####At the end of the course: - A field to choose from a list of value: category in our forms - The Summary tab shows the label of the selected - 2-level list of value
### Vocabularies #### Concepts - List of items used in the application - Id: Code of the reference. - Value: Label displayed in the UI - Obsolete: Hides the value for user selection, but is being kept for data integrity reasons. - Order: Integer used in some widget to order the list - Stored in the database (one table per vocabulary) - Existing vocabularies can be managed in the admin center by administrators or members of the power users group - Can be used with the Vocabulary, Chained vocabulary or Single directory suggestion widgets
### Vocabularies #### Several kinds of vocabularies - Simple: Can be used alone or as a parent vocabulary - Child: Entries are linked to a simple vocabulary - Hierarchical: Both parent and child entries are defined at the same time - Unlimited levels using Studio with the Single directory suggestion widget, 2 levels max with the Vocabulary or Chained vocabulary widgets.
### Vocabularies #### Vocabulary creation policy - Vocabularies are created at server start - On missing columns: The first time, then only after table structure change - Always: At each server restart (good for debug, not recommended for production) - NeverÂ

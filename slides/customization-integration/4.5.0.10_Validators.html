# Nuxeo training - JSF / Seam validators
### Context

Filling metadata is great, but ensuring they are correctly formatted is even better. Let's see how we can deal with it.

###What we want to do ####At the end of the course: - A validator to check the ISBN is made of numbers only
### Seam beans #### Concepts - Schemas store the metadata without controlling it first - To ensure they are properly formatted, a validator can be added - Seam beans are used in JSF templates to call server side logic from the UI: - #{mySeamBean.property} - #{mySeamBean.method()}
### Seam beans #### Annotations ##### Name - The @name(myBeanName) used before the class declaration defines the bean name ##### Scope - Seam beans are instanciated for every user - Beans are destroyed depending on their scope (@scope annotation) - Stateless: destroyed immediately - Event: one call - Page: all calls within a page - Conversation: destroyed by the developer - Session: destroyed at user logout - Application: shared by all users
### Validators #### Syntax - Validators are beans with: - A specific signature - Throwing a ValidatorException #### Example MyValidatorBean
### Seam beans and validators #### Deploying your beans - Beans are deployed if Seam finds a seam.properties file in the src/main/resources folder #### Calling a validator - In Studio, use the validator field (found in widgets) - Call your validator using #{myBeanName.myMethod}

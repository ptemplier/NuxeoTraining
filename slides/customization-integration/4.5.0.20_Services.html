# Nuxeo training - Services
### Context

The code growing in our beans cannot be tested,
and becomes an expanding threat. What can we do then ? Put it in a service.

###What we want to do ####At the end of the course: - Move the logic from our bean to a service
### Services #### Beans VS services

##### Beans - Belong to the UI layer - Cannot be tested easily - Their role: act as an interface - Send UI information to the service for treatment - Send return values to the JSF templates for display ##### Services - Hold the business logic - Can easily be tested - Can provide extension points for further configurability
### Practice #### Services ##### Create a service - Create a new component using Nuxeo IDE's template and expose it as a service - Refactor your code to move out the logic from your bean to your service

# Nuxeo training - Document types
###Context

Now that you have collected the users needs, you understand that in order to manage a book library properly, specific metadata needs to be saved. So you decide to start by creating corresponding document types.

###What we want to do (1/2) ####At the end of the course: - A custom document type: Book - Using common metadata - Title - Containing also custom metadata - ISBN (string) - Author (Complex, multivalued) - publicationDate (date) - rating (int) - We won't display / edit the metadata yet!
###What we want to do (2/2) ####At the end of the course: A BookLibrary document type
Can contain `Book` documents
### Document type ? ####Defined by - a unique name, eg: File, Note or Book - a set of schemas to define its storage structure  - a set of facets which defines the document behavior or capabilities - eg: Folderish, Versionable, Commentable - its parent type : every document type you create inherits from another one - Schemas and facets are both inherited - Base document type in Nuxeo is `Document`, all document types inherit at least from it - Container types: Where the document type can be created
### Document type definition ####UI properties - Label - Category: Where the document will be displayed at creation time - Icons
### Exercise #### Document type creation - Create a new document type, named `Book`. - Keep the default settings and set the following properties: - Label: Book - Extends `Document` - Category: Document - Icons: files/book16.png (small icon) & files/book32.png (large icon) - Add the `Versionable` facet - Add the `Publishable` facet - Commit your changes. - Deploy your changes using hot reload and see if you can create a book.
### Context

The "Best books" library feels that the Book document type will not be enough to organize their collection. Instead of using folders, they want to implement a somewhat similar document type called BookLibrary which should be able to contain books. No problem, you say!

### Document type definition ####The folderish facet - Any folder in Nuxeo is a document type having the "folderish" facet - Applying the "folderish" facet to a document type gives access to: - `Accepted children types`: Which document type(s) can be created in it - `Show create child action`: To display the button allowing the creation of a new document in it - `Show view tab`: Show the "summary" tab, normally hidden by default for folderish document types
### Exercise #### Folderish document type creation - Create a new document type, named `BookLibrary`. - Keep the default settings and set the following properties: - Label: Book library - Extends `Folder` - Category: Collaborative spaces - Icons: files/bookLibrary16.png (small icon) & files/bookLibrary32.png (large icon) - Container types: Folder, Workspace - Accepted children types: Book - `Folderish` facet - Commit your changes. - Deploy your changes using hot reload. - See if you can create a book library, and a book in it.
###Context

Now that their custom document types have been created, the "Best books" library realises that they do not contain the correct fields for them to fill metadata in. Oops. It's time to customize their structure…

### Document types - Schemas ####Defined by - a unique name - a prefix (a kind of namespace or short name) - A set of different fields/metadata  #### Metadata concepts - Each metadata has: - A name  - A type  - Metadatas can have the same name as long as they are located in different schemas - To get or set a metadata Nuxeo uses a unique id, called a `xpath` - A xpath is represented as `schemaname:metadataname`
###Schema metadata types - Common metadata types are: - String: Text - Integer: Integer number - Floating point: Floating number - Blob: Binary file - Boolean: Boolean value - Date: Well, a date - Complex: See below - Complex type: composed of several fields that can have the same or different types.
For instance an `author` complex type could contain: - name (string) - first name (string) - Each field can be multivalued
### Schema creation #### What fields do I need to add? - When you create a document type, Nuxeo imports: - Default schemas: dublincore (title, description, author…), etc. - Inherited schemas: Depending on what your document type extends #### How / where can I add more? - Directly into the `schema` tab of your document type - By creating a schema in `Content Model` > `Schemas` and adding it into the document type definition; makes it reusable for other document types.
### Practice (1/2) #### Schemas - Edit the `Book`'s schema. Let’s say that, for now, a Book is defined by:  - Its ISBN: `isbn` (string)  - Its publication date: `publicationDate` (date)  - A `rating` (int) - Its `author` (complex, multivalued), composed of: - A `name` (string) - A first name: `firstName` (string) 
### Practice (2/2) #### Try your changes - Commit your changes - Deploy your changes using hot reload - Create a book: Can you see the fields you just created?

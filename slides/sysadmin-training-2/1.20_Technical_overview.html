### Nuxeo – The Platform #### Standard technologies - 100% Java based - Based on Java standards: Java, OSGi, JTA, JAAS, JMS, EJB3, JAX-RS, JAX-WS, CMIS, … - Uses many open source frameworks / libraries: JBoss Seam / Hibernate / Apache commons - Can run in any Servlet container, Java EE Application server, bare JVM or OSGI container, typically: Tomcat, JBoss, Jetty #### Easy to customize, extend and plug - No need to rewrite the source code - Offers content oriented APIs - Ability to deploy and integrate in complex environments
###Bundles - 1 bundle = 1 Nuxeo plugin/addon = 1 JAR - Located in nxserver/bundles or nxserver/plugins - Configuration/Extension/Customization Mechanism: Extension Point/Contribution - Loaded by Nuxeo Runtime - More than 220 bundles in Nuxeo CAP

###Configuring and Extending Functionalities #### Extension Points - Functionalities are built into "services" - Services expose extension points to make them configurable through XML - Allow to separate configuration from code => Makes upgrading easy - A bundle with XML configuration can be generated easily with Nuxeo Studio #### Creating New Functionalities - New functionalities are added through bundles containing code / services - Nuxeo IDE helps in generating them

##About Nuxeo Core: Advanced Content Repository
###Nuxeo Document ####Not a simple « file ». ####An instance of a Document type (File, Note, Folder, Invoice, Contract…) having - a unique name, eg: File, Note or Book - a set of schemas to define its storage structure  - a document can have many schemas, schemas can be applied to many document types - a set of facets which defines the document behavior or capabilities - a parent type: the document type will inherit its schemas and facets - a lifecycle (ie: draft -> validated -> archived)
###Nuxeo Document ![doc](../customization-integration/img/installation_demo/whatisadoc.png)
### Repository backend ####Nuxeo Visible Content Store - Dedicated native SQL storage - Provides clean SQL mapping (ie: usable by a DBA or a BI tool) - Multiple database vendor support (PostgreSQL, Oracle, MySQL, MSSQL, H2, Derby) ####Efficient storage - Storage adapters - Meta-data and structure: SQL DB - Binaries: Filesystem, SQLDB, S3 ... - Snapshotable, automatic deduplication, … - Support hot backup and GC

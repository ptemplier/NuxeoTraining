Nuxeo Platform - elasticsearch Configuration

### Context
elasticsearch has now to be linked to the Nuxeo Platform.
### elasticsearch Configuration
First thing to do is to define elasticsearch's cluster name and url.
#### Edit the es config file - sudo nano /etc/elasticsearch/elasticsearch.yml - Uncomment and change the cluster.name property's value - Restart the elasticsearch service: sudo service elasticsearch restart #### Update the nuxeo.conf file - elasticsearch.addressList=localhost:9300 - elasticsearch.clusterName=your-cluster-name
### elasticsearch Configuration
Default fulltext configuration handles english language. It can be changed to french through a custom configuration template.
#### Create a custom template - Make a copy of the templates/custom folder for your custom template - Ensure that it deploys the elasticsearch-config.xml.nxftl into the nxserver/config folder - Include it in your nuxeo.conf file. This template should be deployed after all others
### elasticsearch Configuration
When the template is ready, you can test it easily.
#### Test the text analysis curl -XPOST 'http://localhost:9200/nuxeo/_analyze?analyzer=fulltext&pretty' -d {"C'est génial"} #### Drop the es index curl -XDELETE 'http://localhost:9200/nuxeo/' #### Restart with the updated configuration ./nuxeoctl restart #### Test the text analysis again curl -XPOST 'http://localhost:9200/nuxeo/_analyze?analyzer=fulltext&pretty' -d {"C'est génial"}
### elasticsearch Configuration
Time for the final testing and reindexing.
#### Test the text analysis again curl -XPOST 'http://localhost:9200/nuxeo/_analyze?analyzer=fulltext&pretty' -d {"C'est génial"} #### Reindex the documents - Go to Admin > Elasticsearch > Admin - Re-index documents matching the SELECT * FROM Document query

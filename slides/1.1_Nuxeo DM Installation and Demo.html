to launch Nuxeo without the Gui

## Document Management Tab
###What is a document ? ####Not just a « file ». - Set of metadatas (Title, Creation date, File, ...) - Typed metadatas (string, date, content, complex types, ...) ####An instance of a Document type (File, Note, Module, Lesson, Invoice, Contract, …) - Is defined by a set of metadata grouped by schema - Has a specific lifecycle (ie Draft->Valid->Archived) - Can be inherited (Module extends Folder) - Has facets (Folderish, Versionable, Commentable, ...)

###DM Domain - Sections, workspaces and templates are grouped in domains - A domain can be considered as a mini-application in Nuxeo or a root in the application. - Typically you can use domains to open the application to two or more very different populations of users - For instance, you can have an "intranet" domain and an "extranet" domain - Domains are can be totally independent or not, depending on the access rights you set on each domain. - Nuxeo Documentation http://doc.nuxeo.com/x/XoWo
### DM Domain default structure ####Workspaces A Workspace is for a group of users that work together, usually preferences (security, notifications) are set on Workspaces ####Folders Inside a Workspace, to organize content #### Sections To publish content ####Templates pre-initialize a Workspace structure.
###Access Rights (1/3) ####Available for - All "Folderish" documents - "Manage everything" right http://doc.nuxeo.com/x/UYEk
###Access Rights ####Manage tab, Access rights ![Access rights](img/access-rights-interface.png)
###Access Rights (2/3) ####Default permissions - Read - Delete (+ Read) - Write (+ Delete + Read) - Manage Everything
Some permissions are composed of sub permissions. See http://explorer.nuxeo.org/nuxeo/site/distribution/current/viewExtensionPoint/org.nuxeo.ecm.core.security.SecurityService--permissions for existing permissions.
####Access Control Entry - ACE (3) - Triplet - Principal (user/group) - Permission - Grant/Deny - For instance: `members - Read - grant`
###Access Rights (3/3) ####Acces Control List (ACL) - List of ACE - ACEs on the top overrided by the bottom ones ####Access Control Policy (ACP) List of ACL applied to a document - Inherited ACL (1) - Local ACL (2) #### Bloking inheritance Technically `Everyone - Deny - Everything` apply on the first ACE of the local ACL.
###Versioning ####Don't want to copy and rename - On Edit tab while saving - Skip version increment - Increment minor version - Increment major version - Versions available - History tab, Archived versions - View versions - Restore - Compare (With the nuxeo-diff addon)
###Publication #### Document published - from workspaces (Publish tab of a document) - into Sections ![publication](img/publishing-tab.png) #### Access Right policy example: - Section: `Read` to external people - Workspaces: `Read` and `Write` to internal members
###Behind the publication mechanism #### Document publication - Version of the document created - `proxy` document created - A proxy is a document that reflect the pointed document. (Like a pointer in C language) - The proxy points to the versionned document in a Section
###Publication Workflow #### Built-in publishing workflow - User with only Read permission on the Section needs approval from section manager to validate the publishing - User with the Write permission on the Section will publish without approval - Permission for ability to publish managed from the section #### Publishing targets - Preferences of publishing targets managed from the workspace in the Manage tab ![Publication workflow](img/publishing-rights.png)
###Searching the Nuxeo Platform ####Full text query engine - Designed to recognize all the specific parameters the application provides - Documents are indexed using their title, description, content (note field, attached files), and metadata. #### Several ways to search the content of the application - Simple search, can be accessed from any site page - Advanced search - Faceted search (Document Management module only) - Search form can be customized in Studio ![Advanced search link](img/search-box.png) ![Advanced search form](img/DM-advanced_search_filter.png)
###Faceted Search - Easy way to browse the content of your application - By creating filters on the content of the application - You can filter content using: - Keywords (full text search based on stemming) - Dates (creation and last modification dates) - Author - Documents categorization (nature, coverage, subjects of documents) - Location of the documents in the folders - Faceted search is available on the Document Management module of the Platform, on the left hand side of the page http://doc.nuxeo.com/x/YIEk
###Fulltext search - Any text metadatas indexed - Office, pdf, text, html files are full-text indexed
thanks to a text extraction from the document itself
- Relies on the database (Oracle, Postgres or MSSQL Server) full text capabilities - Linguistic support
Searching for the word "satisfies", will match the documents containing "satisfy"
- Optimized for searching
###Other document services - Comments - Annotations - Tags - Doc history - Alerts - Workflows - Forums - Pictures - Email folders http://doc.nuxeo.com/x/MoEk

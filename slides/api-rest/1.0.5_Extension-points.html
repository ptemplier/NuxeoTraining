# Nuxeo training - Bundles and extension points
### A few words about Nuxeo design concepts #### Nuxeo runtime - The Nuxeo runtime is a small core component that: - Deploys bundles (modules) to add functionalities to the platform - Generates the web application that will be used by the server (Tomcat or JBoss) #### Bundles - Every Nuxeo module is a bundle - A bundle: - Is a jar file - Defined in the META-INF/MANIFEST.MF - Has a unique ID (bundle-symbolicName) - Declares a set of components - Exposes extension points that allow to configure or extend them Extension point are inspired by the Equinox project (OSGI implementation used in Eclipse)
### Inside a bundle #### MANIFEST.MF Contains OSGi entries: 
Bundle-ManifestVersion: 1
Bundle-Name: Your bundle name
Bundle-Version: 1.0.0
Bundle-Vendor: Your company name
The unique name representing your bundle: 
Bundle-SymbolicName: a.unique.name
Nuxeo specific: Component registration: 
Nuxeo-Component: OSGI-INF/one-component.xml, OSGI-INF/another-one.xml
#### Notes  - Be careful with spaces and commas - Pay attention to your SymbolicName - it has to be unique. A good choice is the package you are providing - Always leave a new line at the end of the file
###Components #### Concepts - Defined in XML descriptors - Declared in the MANIFEST bundle - Provides: - One or more services - One or more Extension Points - One or more Extension Points xml contribution
###Extension points #### Concepts - Contribute configuration to existing EPs  - Contribute code and behavior to existing EPs  - Other components extend / customize your service #### What we code ##### In Nuxeo studio - Contributions to implement business logic  ##### In Nuxeo IDE - Contributions - Java components that can be extensible too

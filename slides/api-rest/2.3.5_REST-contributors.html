# Nuxeo Training - REST Contributors
### Context

One of the strengths of the Nuxeo REST API is its modularity.
We could use it to get our own cuztomized objects, and that's exactly what we will do !

### What we want to do #### At the end of the course - Contribute custom contributors to get the navigation breadcrumbs of a document - Optionally, use a parameter to limit the number of levels returned
### Contributors #### Concepts - Java classes defining custom information to be returned - Belong to a category - When making a REST call, REST contributors categories can be added using the X-NXContext-Category header
### Contributors #### Concepts - Java classes implementing RestContributor - Defining custom information to be returned - Belonging to a category - REST contributors categories can be called using the X-NXContext-Category header - Custom properties show up in the context parameters returned by the call - JSON is written using the org.codehaus.jackson library
### Contributors - Example 
public class HasWritePermissionContributor implements RestContributor {

  @Override
  public void contribute(JsonGenerator jg, RestEvaluationContext ec) {
    // RestEvaluationContext contains the doc, the URL parameters, the headers
    DocumentModel doc = ec.getDocumentModel();
    
    // The contributor has to write some JSON to be returned
    ObjectNode myJsonObject = buildMyObject(doc);
    jg.writeObject(myJsonObject);
  }

  private ObjectNode buildMyObject(doc) {
    // JSON needs to be put in an ObjectNode, let's create it
    ObjectMapper o = new ObjectMapper();
    ObjectNode myObjectNode = o.createObjectNode();

    // Get the information and put it in the ObjectNode
    // ObjectNodes can be nested 
    CoreSession documentManager = doc.getCoreSession();
    boolean hasWritePermission = documentManager.hasPermission(doc.getRef(), "Write");
    myObjectNode.put("hasWritePermission", String.valueOf(hasWritePermission));

    return myObjectNode;
  }
}

### Context

Now that the code is ready, we have to call it somehow.

### Contributors #### Declaring your contributor ##### In Nuxeo Studio - Go to the Advanced settings, XML extensions menu ##### Or in eclipse - Add a XML file in the src/main/resources/OSGI-INF/extensions folder - In the src/main/resources/META-INF/manifest.mf file, declare your XML extension:
Nuxeo-Component: OSGI-INF/extensions/myFile.xml #### XML extension code 
<extension target="org.nuxeo.ecm.automation.io.services.contributor.RestContributorService" point="contributor">
      <contributor name="mybreadcrumb" class="org.nuxeo.sample.MyBreadcrumbContributor">
        <category>mybc</category>
      </contributor>
</extension>

### Contributors #### Parameters RestEvaluationContext provides a way to retrieve parameters ##### URL parameters String myParam = ec.getRequest().getParameter("myParam")); ##### Header parameters String myHeader = ec.getHeaders().getRequestHeader("myHeader").get(0); 
@Override
public void contribute(JsonGenerator jg, RestEvaluationContext ec) throws ClientException, IOException {
  DocumentModel doc = ec.getDocumentModel();
  String myParam = ec.getRequest().getParameter("myParam");
  String myHeader = ec.getHeaders().getRequestHeader("myHeader").get(0);

  ObjectNode myJsonObject = buildMyObject(doc, myParam, myHeader);
  jg.writeObject(myJsonObject);
}

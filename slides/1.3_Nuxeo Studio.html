###Nuxeo and Studio - behind the magic - Nuxeo platform: modular with plugins and the extension point mechanism
Nuxeo Platform, uses the pattern of exension points. That means that each time Nuxeo developper wants to implement a feature that can be configured by other developpers when being used, they code a «hook» in the middle of the feature. A hook in the code is a sort of «indirection». The framework, when finding a reference to a hook in the code, reads the name of the hook, and fetches in the «extension library» what values were contributed for this given hook. Then the implementation code of the feature will leverage those values to do what expected.
- Without Studio, xml extension points contribution
When an external developer wants to adapt a Nuxeo EP based solution, he has to contribute to the Extensions library. The Nuxeo EP framework allows to contribute to this extensions library through XML files packaged as plugins in jars (java packaging format) files.
- Existing extensions http://explorer.nuxeo.org/nuxeo/site/distribution/current/listExtensionPoints
There are hundreds of them.
- Meta-model on top of extensions
Studio is a meta-model on top of extensions, which are more unitary. For instance, when configuring a document type in Studio, behind the scene, Studio will generate multiple extensions: for core type declaration, forms, lifecycle, ... To see what extensions Studio generates, you can go to Settings and Versioning>Project Settings > Manifest and load the extensions.xml file.

##Exercise
###First Studio Exercise #### Connect - Create a trial account and access Nuxeo Studio [https://connect.nuxeo.com/nuxeo/site/connect/trial/form](https://connect.nuxeo.com/nuxeo/site/connect/trial/form) - Configure your connect account on your Nuxeo DM instance: Admin Center -> Nuxeo Connect #### Studio * Connect to your Nuxeo Studio Account, either with Direct Access URL or from Nuxeo DM (User Administrators > Admin center > Update Center > Studio > Go to Nuxeo Studio link) * Browse the pre-imported project (Default File, Domain, Workspace document type.) * Deploy your studio project on DM (Admin Center -> Update Center -> Nuxeo Studio -> Update) * Check that your project is correctly deployed (you’ll see new columns name, etc...) * Unimport the DM Default Template, and delete objects that were not automatically unimported * Go to Settings and Versionning > Versionning, create a tag called «empty project»: goal will be to be able to switch back to an empty project at anytime of your project

#Nuxeo Training - Structure templates
### Context

The custom documents we created are perfect. The "Best books" library now decides they should be used all over; to do so they would like a simpler domain structure, in which mostly books and book libraries would be created. Easy :)

###What we want to do ####At the end of the course: #### A new structure for the type Domain - Book Library - "Sandbox" (Workspace) - Sections - Internal memos #### Instead of - Sections - Templates - Workspaces
###Structure template #### Concepts - Pre-built structures - «Per document type» basis - Defines what documents have to be generated when a document is created  - Useful to setup the default structure of your repository - Can also define the default set of permissions - Can override the existing structure template for a doc type Ex: Each time you create a Domain, nodes templates, Sections and Workspaces are created
### Structure template #### Choose the type The created structure will apply to the chosen document type ![Choose the type in Studio](img/structure-template-choose.png)
###Structure template #### Configuration - The structure of Domain will be overriden with this one - This structure will be used - Each time a document of type Domain will be created - At startup if the database is empty - Existing documents of type Domain will keep their old structure
###Practice - Structure template - Modify the default structure of the type Domain to have: - BookLibrary, - A Workspace named "Sandbox" - Sections, - A Section named "Internal memos" - Commit your changes - Deploy your changes - Create a domain and see what appears

## Document management tab - Document concept and creation
###What is a document? ![doc](img/whatisadoc.png)
###What is a document? ####Not just a « file ». ####An instance of a Document type (File, Note, Folder, Invoice, Contract…) - Is defined by a set of information called `metadata` - Metadata can be filled automatically (creation date, name of the file…) or manually (description, subject...) - Metadata is grouped into `schemas`. A document can have many different schemas, and schemas can be shared between document types. - Has a specific `lifecycle` (ie Draft->Valid->Archived) - Has `facets` defining its behavior and what you can do with it (Folderish, Versionable, Commentable...) - Can inherit facets and schemas from another document type (eg: Module extends Folder)
###Creating a new document manually - Step 1/3 - `New document` - Select the document type - The list of available document types depends on the container document type ![Available doc types](img/available-documents-workspace.png)
###Creating a new document manually - Step 2/3 - Choosing a doc type #### Main doc types - `File`, document with an attached file - `Note`, document with a rich text - `Workspace`, a folder where users can collaborate - `Folder`, a limited folder used to organize documents inside a workspace #### More ... - `Picture` - `Picture book` - `Website` - etc http://doc.nuxeo.com/x/S4Ek

## Document management tab - Document publication
###Publishing a document - Document publish tab #### Concepts - Documents are meant to be published when a document version is good enough to be released - Documents are published into sections - Documents located into sections cannot be modified ![publication](img/publishing-tab.png)
###Publishing a document - Behind the publication mechanism #### Publishing a document - Creates a version of the document if necessary - Creates a `proxy` document into the selected section - A proxy is like a shortcut that points to the versioned document
###Publishing a document - Publication workflow #### Built-in publishing workflow - User with only Read permission on the Section needs approval from section manager to validate the publishing - User with the Write permission on the Section will publish without approval - Permission for ability to publish is managed from the section #### Publishing targets - Preferences of publishing targets are managed from the workspace in the `Manage` tab ![Publication workflow](img/publishing-rights.png)

## Document management tab - Document search
###Searching the Nuxeo Platform #### Several ways to search the content of the application - Simple search, can be accessed from any site page - Advanced search - Faceted search (Document Management module only) - Search form can be customized in Studio - Search results are displayed depending on user rights ![Advanced search link](img/search-box.png) ![Advanced search form](img/DM-advanced_search_filter.png)
###Faceted Search - Easy way to browse the content of your application - By creating filters on the content of the application - You can filter content using: - Keywords (full text search based on stemming) - Dates (creation and last modification dates) - Author - Documents categorization (nature, coverage, subjects of documents) - Location of the documents in the folders - Faceted search is available on the Document Management module of the Platform, on the left hand side of the page http://doc.nuxeo.com/x/YIEk
###Fulltext search - Any text metadata indexed - Office, pdf, text, html files are full-text indexed
thanks to a text extraction from the document itself
- Relies on the database (Oracle, Postgres or MSSQL Server) full text capabilities - Linguistic support
Searching for the word "satisfies", will match the documents containing "satisfy"
- Optimized for searching

## Document management tab - Access rights management
###Access rights ####Are available - On all `Folderish` documents - For users being granted the `Manage everything` right - Using the `Manage` tab, `Access rights` subtab ####Concepts - Access rights are inherited from their parent document (1) - Local rights can be granted or denied to override inherited permissions (2 & 3) - Inheritance can be blocked for specific rights to be defined on a document more easily (1) http://doc.nuxeo.com/x/UYEk
###Access rights ![Access rights](img/access-rights-interface.png)
###Access rights ####Default permissions - Read - Version (manage versioning + Read) - Write (+ Delete + Read) - Remove (usually denied to users having write permission to avoid them deleting documents) - Manage Everything ####Default user groups - Members: includes everyone, have read access by default - Power users: functional admins; can create users and groups, vocabulary entries - Administrators: can manage everything ####Access rights priority - Local rights have the prority over inherited rights - Denied rights have priority over granted rights
Some permissions are composed of sub permissions. See http://explorer.nuxeo.org/nuxeo/site/distribution/current/viewExtensionPoint/org.nuxeo.ecm.core.security.SecurityService--permissions for existing permissions.

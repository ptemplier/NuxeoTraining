Nuxeo - Technical overview =======================
###Complete documentation available on doc.nuxeo.com - Technical documentation center - Installation and Administration
###Core technologies #### Standard technologies - Based on Java standards - Uses many open source frameworks / libraries : JBoss Seam / Hibernate / Apache commons - Can run in any Servlet container, Java EE Application server, bare JVM or OSGI container, typically: Tomcat, JBoss, Jetty #### Easy to customize, extend and plug - No need to rewrite the source code thanks to the bundle and component system providing extension points - Offers content oriented APIs : Webdav, CMIS (Apache chemistry), WSS (Microsoft sharepoint services)
###Bundles - 1 bundle = 1 Nuxeo plugin - located in nxserver/bundles as jar files - extension mechanism: Extension point (xml files) - loaded by Nuxeo Runtime - more than 200 plugins in Nuxeo CAP
###Extension points #### What are they used for? - To configure services and components (XML contributions) - To extend services and components (Java code or scripting) - To dynamically bind components together #### Where and why? - Nuxeo Platform uses extension points everywhere - Almost every service exposes extension points - Makes configuration easy (no need to touch the source code) - Allows « clean » contributions easy to maintain (XML) - Allows the use of Nuxeo Studio!
###Administering #### Easy to configure and maintain - The `nuxeo.conf` file contains properties to configure many things - Editable directly with a text editor or from the Admin center. - Many databases supported - Clean SQL mapping, easy to administer and retrieve information - Hot database backup support - Hotfixes can be installed directly from the admin center using the dedicated interface

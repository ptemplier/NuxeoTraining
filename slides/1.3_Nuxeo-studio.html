# Nuxeo training - Introduction to Nuxeo studio
##Understanding Nuxeo studio
###Nuxeo studio ####Nuxeo studio is: - A visual tool for configuring Nuxeo Platform based applications - Available as an online service (SaaS) - A delivery and deployment channel through Nuxeo Update Center #### With Nuxeo studio you can: - Configure and adapt your application - From document typology to business rules enforcement - Without needing development work - Save time and money, by focusing on your business needs - Ensure long-term maintenance of your application thanks to the target platform deployment option
###Nuxeo studio - How does it work? - Direct access via Connect portal - 30 days free trial - Provides a jar file with your project configuration - studio produces no compiled code, XML only - Development tools integration (Maven and Nuxeo IDE)
###Nuxeo studio - Main features - Branding (Theme configuration) - Document types and forms configuration - Content views and search features - Lifecycle, workflows, automation, notifications - Roles, permissions, users and groups - Ability to import pre-built Application Templates
###Practice Nuxeo studio #### Connect - Create a trial account and access Nuxeo studio - Configure your Nuxeo Connect account on your Nuxeo instance: Admin Center > Nuxeo Connect
##Nuxeo studio - Versioning
###Nuxeo studio - Versioning #### Concepts - Save your project state - Revert to a previous state anytime - Release versions and deploy them easily - Simplified versioning tool: No branching and no conflict management
###Nuxeo studio - Versioning #### Three different stages - Commits - Preferably used for small changes - Do not hesitate to commit often so you can revert a change if needed without losing too much - Tags - When you feel the version is complete enough - Any commit can be tagged - Tags remain available even if you revert to a commit done before the tag - Snapshots - When you feel the version can be released for production - Snapshots can be downloaded and installed directly from the admin center
###Nuxeo studio - Versioning #### Management - Define your commit style from Settings and versioning > Project settings - On save: Automatic commit each time you press save on a screen - On build: When you create a tag or a snapshot- Manual: Only when you choose to - No branching and no conflict management implementation
###Practice Nuxeo studio (1/2) #### Familiarize with studio * Connect to your Nuxeo studio Account, either with Direct Access URL or from Nuxeo DM
`Admin center` > `Update Center` > `Studio` > `Go to Nuxeo studio` link * Unimport the DM Default Template
`Settings and versioning` > `Application templates` > `Nuxeo DM default configuration` > `Remove` * Set your project commit style to manual
`Settings and versioning` > `Project settings` * Go to `Settings and Versioning` > `Versioning`, create a tag called `emptyProject`: Goal will be to be able to switch back to an empty project at anytime of your project * Create a new branding called `bestBooksTheme`
`Project` > `Branding` * Leave the default values except for the logo image: files/bestBooksLogo.png * Save
###Practice Nuxeo studio (2/2) #### Test your changes * Deploy your studio project on DM
`Admin Center` > `Update Center` > `Nuxeo studio` > `Update` * Check that your project is correctly deployed (look at the logo on the top left)

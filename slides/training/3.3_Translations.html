# Nuxeo training - Translations
### Context

The general manager of the "Best books" library is preparing to open another library abroad, which of course will be using Nuxeo too.

He asks you to prepare everything for the application to be translated when the new library opens.

### What we want to do #### At the end of the course - Give translations to "Book" life cycle states - Change the name of the "Home" tab
###Translations #### Concepts - Nuxeo can be translated in any language - Default set of translations - Provided by Nuxeo: english and french - Provided by the community: many others:) - Translations are files located in each bundle - At server start, Nuxeo concatenates them
### Translations #### More details about translation files - Translations are integrated into `messages_LANG.properties` files - LANG being the ISO code (ex: fr) - Mapping is done using `label = my translation` - Files use unicode encoding, be careful with accents! - Existing labels can be reused and overriden - No specific convention regarding the translation labels Example: 
inLibrary=Dans la biblioth\u00e8que
borrowed=Emprunt\u00e9
label.main.tab.home=Accueil
#### Where are the existing translation files? nxserver/nuxeo.war/WEB-INF/classes/messages_LANG.properties
### Translations #### How can I add my own? - In studio - Upload your custom messages_LANG.properties files in the resources menu - Be careful: some labels need to be explicitly marked as "translated" or "localized" for the translation to become effective - Programatically - http://doc.nuxeo.com/x/dAQz
### Practice #### Translations - Change the name of the "Home" tab - Give translations to "Book" life cycle states - Commit and deploy your changes - Check that your labels have been translated

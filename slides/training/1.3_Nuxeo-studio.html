# Nuxeo training - Introduction to Nuxeo studio
###Context

The "Best books" library has decided to implement Nuxeo in order to manage its book collection.

And YOU will be in charge of their project.

Specifications are being written at the moment. In the meantime, you need to familiarize yourself with the tools you are going to use.

##Understanding Nuxeo studio
###Nuxeo studio ####Nuxeo studio is: - A visual tool for configuring Nuxeo Platform based applications - Available as an online service (SaaS) - A delivery and deployment channel through Nuxeo Update Center #### With Nuxeo studio you can: - Configure and adapt your application - From document typology to business rules enforcement - Without needing development work - Save time and money, by focusing on your business needs - Ensure long-term maintenance of your application thanks to the target platform deployment option
###Nuxeo studio - How does it work? - Direct access via Connect portal - 30 days free trial - Provides a jar file with your project configuration - studio produces no compiled code, XML only. Your local Nuxeo instance connects to Studio and deploys the configuration made - Development tools integration (Maven and Nuxeo IDE)
###Nuxeo studio - Navigation and features #### Project information Get the essential information right away.
###Nuxeo studio - Navigation and features #### Project menu Configure your application by adding new components.
##### Buttons - New : add a new element - Import : import pre built application templates - Download : retrieve your project as a jar file - Commit : commit your changes into studio's versioning tool ##### Sub menus - Resources : upload new elements (images, translation files...) - Branding : define your logo, colors... - Content model : create your document types, schemas (metadata)... - Workflows - Listing and views : customize your documents presentation and navigation - Automation : implement your business logic - Templates : email and document templates creation - Roles and permissions : define your own permissions and default users / groups - Vocabularies : handle lists of pre defined options - Advanced settings : extend your platform even further
###Nuxeo studio - Navigation and features #### Settings and versioning menu Define your target platform and general options there.
- Project settings : define your general settings including the platform target version - Versioning : manage your application versions with this integrated tool - Application templates : import pre built application templates (identical to the import button) - Registries : advanced feature - declare your custom developed elements to use them in studio
###Practice Nuxeo studio (1/2) #### Familiarize with studio * Connect to your Nuxeo studio Account, either with Direct Access URL or from Nuxeo DM
`Admin center` > `Update Center` > `Studio` > `Go to Nuxeo studio` link * Unimport the DM Default Template
`Settings and versioning` > `Application templates` > `Nuxeo DM default configuration` > `Remove` * Create a new branding called `bestBooksTheme`
`Project` > `Branding` * Leave the default values except for the logo image: files/bestBooksLogo.png * Save
###Practice Nuxeo studio (2/2) #### Test your changes * **Deploy your studio project on your local DM instance**
`Admin Center` > `Update Center` > `Nuxeo studio` > `Update` * Activate the development mode to allow use of the hot reload functionality * Restart your instance * Check that your project is correctly deployed (look at the logo on the top left)

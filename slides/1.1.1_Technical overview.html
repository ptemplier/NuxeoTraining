Nuxeo - Technical overview =======================
###Complete documentation available on doc.nuxeo.com - Technical documentation center - Installation and Administration
###Core technologies - Based on Java standards: POJO, OSGI, JCA, JTA, JAX-RS, KSF, etc - Can run in any Servlet container, Java EE Application server, bare JVM or OSGI container, typically: Tomcat, JBoss, Jetty - OSGi-based Bundle and Component system - Highly dynamic plugin model - Great pluggability through extension points - Many open source frameworks / libraries : JBoss Seam / Hibernate / Apache commons - Content oriented APIs : Webdav, CMIS (Apache Chemistry), WSS (MS Windows Sharepoint Services)

###Inside Nuxeo ![nuxeo-layers](img/nuxeo-layers.png)
###Extension points - Extension points are used: - To configure services and components (XML contributions) - To extend services and components (Java code or scripting) - To dynamically bind components together - Nuxeo Platform uses extension points everywhere - Almost every service exposes extension points - Makes configuration easy (no need to touch the source code) - Allows « clean » contributions easy to maintain (XML) - Allows the use of Nuxeo Studio!
###What we code in Nuxeo - Contribution to extension points (XML) => Nuxeo Studio can help you - Event handlers => Studio can help you - Components that expose extension points (XML / JAVA) => Nuxeo IDE can help you - Event listeners => Nuxeo IDE can help you - Seam components / XHTML templates => Nuxeo IDE can help you

###Configuration: nuxeo.conf - The `nuxeo.conf` file contains several properties to configure many things - Editable directly with a text editor or from the Admin center. - Property modification needs a restart - Startup wizard sets values in it - See online documentation about the existing properties
###Configuration templates - Goal: complete a « physical » separation between customer set up and the core product so as to make the product upgrade as simple as replacing a file - The custom configuration can be done using only the `nuxeo.conf` file and additional configuration files in a custom template - See online documentation about it
###VCS : storing information in a relational database - A schema in Nuxeo is mapped to a table in the database (= clean SQL mapping) - A DBA can easily query, dump, run statistics on the database in the usual way - Multiple database vendors support: PostgreSQL, Oracle, MySQL, MS SQL Server, h2, derby - A DBA can set up additional indexes (this is fine tuning, not needed in common use cases) - Migration, updates, ... should be very simple - See online documentation about it
###Backup / Restore - Hot backup supported - Backup = Database snapshot then file system backup and server configuration
###Importing data into Nuxeo - CMIS, WebDAV, WSS, Nuxeo Drive - RestAPI - Simple POST - Drag&Drop via browser if HTML5 supported - Batch import framework (nuxeo-platform-importer) - Manages all technical details: transaction management, batch size, logging, … - A lot of examples available - Low level SQL batch insert (fastest, but must know what you are doing)
###Maintenance: install the hotfixes - From the Admin Center > Update Center > Nuxeo software updates, with the mouse! - Read the information, respect dependencies required by the system - Follow the Jira link to know all the details about what the hotfix fixes - How bugs are identified between two hotfixes? - Nuxeo internal testing - Customers having Silver, Gold or Platinium support that can ask for fixing of a given bug - Community testing
###Exercise - Install the latest hotfix for Nuxeo 5.6 - From the Admin Center - With `nuxeoctl mp-hotfix` - Find where the `nuxeo.conf` file is on your machine (for Windows users you can have a look at this page)

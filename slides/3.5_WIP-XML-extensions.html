# XML extensions
### What we want to do #### At the end of the course - Deactivate the "by coverage" and "by subject" navigations
### XML extensions #### Concepts - Nuxeo exposes extension points almost everywhere - These extension points can be configured using XML (what studio does) - Not all extension points are configurable in studio yet - For these, studio integrates the XML extension functionality
### XML extensions #### How does it work? (1/2)
  • Go to explorer.nuxeo.org
  • Select your platform version
  • Go to the extension points
  • Retrieve the component name (the long one)
  • Search for its extension points
  • Look at the extension point XML source
### XML extensions #### How does it work? (2/2) In studio, go to the Advanced settings > XML extensions menu and add your XML code 
<extension target="org.nuxeo.ecm.webapp.directory.DirectoryTreeService"
  point="trees">
  <directoryTree name="bySubjectsNavigation" enabled="false" />
  <directoryTree name="byCoverageNavigation" enabled="false" />
</extension>
- Target is the component name - Point is the extension point - An optional <require>component name</require> tag can be added at the beginning of the XML extension to tell Nuxeo the extension should be loaded after a specific component (useful for dependencies management)
### Practice #### XML extensions - Create an XML extension to deactivate the "by coverage" and "by subject" navigations - Commit and deploy your changes - Check that these navigations have been removed

Nuxeo Training - User actions and automation chains ========================
Automation chains and operations --------------------------
###Operations
Unitary and reusable operators that implement a particular documentary function, such as "move a document", "update a property", "change the lifecycle state", "transform to PDF", "send as email"...
####An operation is defined by - A unique name - Input contents - Ouput contents - Process (can contain parameters) #### Accepted input and output - Document - List of Documents - Blob - List of Blobs - void
### Example: Follow transition ####For instance, the "Follow Life Cycle transition" - Name: Document.SetLifeCycle - Input: Documents, Document - Output: Documents, Document - Parameter: Transition the name of the transition to follow - Process: - follow the transition in parameter to the input document - returns the modified document
###Automation chains Chains operations - Build business logic by chaining - Output should fit the next operation input - Studio checks operation inputs and outputs - launched from user actions, events, REST calls and workflows
###Automation Chain
###A Chain in Studio - Ordered set of operations - Respecting input and output of operations - Sharing a context in the whole chain, with variables - Some chains are aware of the UI context, some are not. - The chain takes as input the UI Current Document or the document bound to the Event Handler (more later)
###A chain in Studio ![Studio automation UI](img/automation-studio.png)
###Available operations - Categories - Documents - Execution flow - Fetch - Services - etc ...
##Using automation for creating, updating and moving data
### Fetching Documents ####Initialize the input of your chain - by Id or Path - Fetch > Document operation - Id: aced8d8f-4780-45e0-9605-b0d96119bb89 - Path: /default-domain/workspaces/ta-slides-and-docs - by a NXQL query - Fetch > Query operation - by the contextual document (default fetch)
which is either the current document in the UI or the document bound to the event, depending on the context of execution of the chain
- Some specific UI contextual document - Fetch > UI CurrentWorkspace, - Fetch > UI Current Domain, - ...
###XPath: handle metadata values - xpath to identify metadata property - in operation parameters Document > Update Property - ex: dc:title (dublincore title), dc:subjects, uid:uid - schema_prefix:property - Get properties from Mvel Document object Document['dc:title']
Can be used with Document object in scripted expressions of operation parameters: Document[dc:title] will return the title of the document that was in input of the operation (dc is the prefix of the Dublincore schema)
###Useful operations - Document > Create: creates a document right below the input one - Document > Update Property, Document Update Properties - Document > Move - Document > Copy, Document > Publish Document - Document > Delete
###Inject dynamic values with MVEL - In operation parameters: @{mvel script evaluated} - Objects browser in Studio, from each operation parameters
will help you use the good syntax
- Object Fn, the Document Wrapper, the Date Wrapper. - MVEL http://mvel.codehaus.org/Getting+Started+for+2.0
###Fn Object Collection of useful helpers
available in automation parameters when using the script ( @{ script_expression})
- getNextId(string)
returns the next sequence item for the given string (enables to maintain in parallel multiple sequences)
- getDate()
return a Date object from a date property
- getVocabularyLabel(VocabularyName, KeyName)
returns the label for a vocabulary entry. (ex: useful when automatically updating some fields)
- getEmailsFromGroup(String groupName), getEmails, ...
useful helpers for resolving users, emails, etc, ...
